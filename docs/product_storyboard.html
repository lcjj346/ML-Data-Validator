ML Data Validator
Clean data that never leaves the building.

A trainable validation engine that learns what your data should look like, flags what doesn't belong, and suggests the fix — running entirely on one local machine. No cloud. No API keys. No patient record ever leaves the room.

Act I · The problem

01

Friday, 6:14 pm. Two thousand rows to check by Monday.

Mei Lin coordinates a clinical study. Every week the export lands in her inbox: patient records typed by six different people across three sites. Her review tool is Excel, her method is scrolling, and her error rate is whatever her eyes miss after hour two.

02

The obvious shortcut is forbidden — and rightly so.

Paste it into a chatbot? That's a PDPA breach the moment she hits enter. Patient data cannot touch a third-party server — not for cleaning, not for anything. Cloud data-quality tools are off the table by policy, not by preference. The constraint isn't a limitation of her team. It's the spec.

✗ ChatGPT / Claude / cloud APIs  — data leaves the machine
✗ SaaS data-quality platforms   — data leaves the machine
✗ "Just be more careful in Excel" — not a system
✓ What she needs: something smart that runs where the data lives.

Act II · The product

03

One command. One URL. Zero network calls.

ML Data Validator is a local web app: a Python backend and a React interface served from a single port on her own laptop. It binds to localhost by design — other machines on the network can't even see it.

terminal
$ python run.py
ML Data Validator → http://localhost:8000
04

Teach it your data — fifty clean rows is enough.

No rules to write, no schema to configure. Mei Lin uploads one clean sample of her own study data and the engine learns each column's shape: which columns are categorical, what value ranges are normal, what a valid ID pattern looks like. Wards, blood types, dosage codes — any column, any domain.

05

Two seconds later, every problem is on the screen.

The weekly export goes in; a seven-stage pipeline goes to work — deterministic rules first, then known-value matching, learned ranges, typo detection, and a per-column classifier last. Valid cells turn green. Invalid cells turn red, each with the reason and the stage that caught it.

06

It doesn't just point — it suggests the fix.

Typos come back with the intended value attached. One click applies a fix; one button applies them all. Numbers that are simply wrong get flagged without a made-up "correction" — the engine knows the difference between a typo and a bad measurement.

07

Every change is on the record.

Clinical work demands documented data changes. Alongside the cleaned CSV, Mei Lin exports a quality report and an audit log — timestamp, row, column, original value, new value, and whether a human or a suggestion made the change. Compliance isn't an afterthought; it's an export button.
